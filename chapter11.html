A 25 terms

Attention Architecture
A mechanism that allows models to focus on different parts of input when producing output, crucial for Transformers and modern LLMs.
Formula:
Attention(Q,K,V) = softmax(QK^T/√d_k)V
AGI (Artificial General Intelligence) Techniques
A theoretical form of AI that matches or exceeds human cognitive abilities across all domains, not just specific tasks.
Current Status:
• Not yet achieved
• Active research area
• Debate on timeline
• Safety concerns
Alignment Techniques
Ensuring AI systems behave according to human values and intentions, including safety, helpfulness, and honesty.
Methods:
• RLHF (Reinforcement Learning from Human Feedback)
• Constitutional AI
• Red teaming
• Value learning
API (Application Programming Interface) General
A set of protocols and tools that allows different software applications to communicate with AI models and services.
Example Usage:
curl -X POST https://api.openai.com/v1/chat/completions \
-H "Authorization: Bearer $API_KEY" \
-d '{"model": "gpt-4", "messages": [...]}'

B 18 terms

Backpropagation Training
Algorithm for training neural networks by calculating gradients of the loss function and propagating errors backward through layers.
Process:
1. Forward pass
2. Calculate loss
3. Compute gradients
4. Update weights
Bias Evaluation
Systematic errors in model predictions, often reflecting prejudices in training data or unfair treatment of certain groups.
Types:
• Selection bias
• Confirmation bias
• Algorithmic bias
• Historical bias
Batch Size Training
Number of training examples processed together in one iteration. Affects training speed, memory usage, and convergence.
Common Sizes:
• Small (1-32): More updates
• Medium (64-256): Balanced
• Large (512+): Faster training
BERT Architecture
Bidirectional Encoder Representations from Transformers - a revolutionary pre-trained language model by Google.
Key Features:
• Bidirectional training
• Masked language modeling
• Next sentence prediction
• Transfer learning

C 22 terms

Chain-of-Thought Techniques
Prompting technique that encourages models to show step-by-step reasoning before arriving at an answer.
Example:
"Let's solve this step by step:
1. First, identify the key information...
2. Then, apply the formula..."
Context Window General
Maximum number of tokens a model can process in a single interaction, including both input and output.
Current Limits:
• GPT-4: 8K-128K tokens
• Claude: up to 200K
• Gemini: up to 1M tokens
CNN (Convolutional Neural Network) Architecture
Deep learning architecture especially effective for processing grid-like data such as images.
Components:
• Convolutional layers
• Pooling layers
• Fully connected layers
Constitutional AI Techniques
Training method using principles (constitution) to train AI systems to be helpful, harmless, and honest.
Process:
1. Self-critique
2. Revision
3. RL from AI feedback

D 15 terms

Deep Learning Architecture
Machine learning using neural networks with multiple hidden layers to learn complex patterns in data.
Characteristics:
• Multiple layers
• Automatic features
• End-to-end learning
Dropout Training
Regularization technique that randomly sets input units to zero during training to prevent overfitting.
Benefits:
• Reduces overfitting
• Improves generalization
• Acts as ensemble
Diffusion Models Techniques
Generative models that learn to reverse a gradual noising process for high-quality content generation.
Examples:
• DALL-E 2
• Stable Diffusion
• Midjourney
Data Augmentation General
Techniques to artificially expand training datasets by creating modified versions of existing data.
Methods:
• Rotation, scaling
• Noise addition
• Paraphrasing

E 12 terms

Embedding Architecture
Dense vector representation of discrete objects (words, tokens) capturing semantic meaning in continuous space.
Properties:
• Dense vectors
• Semantic similarity
• 256-4096 dimensions
Emergent Abilities Evaluation
Capabilities that appear in large models at certain scales but are not present in smaller models.
Examples:
• Few-shot learning
• Chain-of-thought
• Code generation
Epoch Training
One complete pass through the entire training dataset during the training process.
Training:
• Multiple epochs needed
• Can cause overfitting
• Convergence metric
Ensemble Learning Techniques
Combining multiple models to create a stronger predictor than any individual model alone.
Methods:
• Bagging
• Boosting
• Stacking
• Voting

F 16 terms

Few-Shot Learning Techniques
Ability to learn and perform tasks with only a few examples, without additional training.
Variants:
• Zero-shot: No examples
• One-shot: Single example
• Few-shot: 2-10 examples
Fine-tuning Training
Process of adapting a pre-trained model to specific tasks or domains using task-specific data.
Types:
• Full fine-tuning
• Parameter-efficient (LoRA)
• Instruction tuning
Foundation Model Architecture
Large-scale models trained on broad data that can be adapted to a wide range of downstream tasks.
Examples:
• GPT family
• PaLM
• Claude
• LLaMA
Feed-Forward Network Architecture
Neural network where connections between nodes do not form cycles, information flows in one direction.
Structure:
• Input layer
• Hidden layers
• Output layer
• No feedback loops

G 14 terms

Generative AI Architecture
AI systems that create new content (text, images, audio, video, code) by learning patterns from training data.
Types:
• Text: GPT, Claude
• Images: DALL-E, Midjourney
• Audio: ElevenLabs
• Code: Copilot
Gradient Descent Training
Optimization algorithm used to minimize loss function by iteratively moving in direction of steepest descent.
Variants:
• Batch GD
• SGD
• Mini-batch GD
• Adam optimizer
Guardrails Techniques
Safety mechanisms to prevent AI models from generating harmful, inappropriate, or undesired content.
Implementation:
• Content filtering
• Output monitoring
• Safety classifiers
GPT Architecture
Generative Pre-trained Transformer - autoregressive language model series by OpenAI.
Evolution:
• GPT-1: 117M params
• GPT-2: 1.5B params
• GPT-3: 175B params
• GPT-4: ~1.7T params

H 11 terms

Hallucination Evaluation
When AI models generate incorrect, fabricated, or ungrounded information presented with confidence.
Types:
• Factual errors
• Fake citations
• Non-existent entities
• Inconsistent info
Human Feedback Techniques
Using human evaluations and preferences to improve AI model performance and alignment with human values.
Methods:
• RLHF
• Direct preference optimization
• Human evaluation studies
Hyperparameters Training
Configuration settings that control training process and model architecture, set before training begins.
Examples:
• Learning rate
• Batch size
• Number of layers
• Hidden dimensions
Hugging Face General
Platform and company providing tools, models, and datasets for machine learning, especially NLP.
Services:
• Model Hub
• Transformers library
• Datasets
• Spaces (demos)

I 13 terms

In-Context Learning Techniques
Ability of language models to learn and adapt to new tasks based on examples provided in input context.
Mechanism:
• Pattern recognition
• No gradient updates
• Emerges at large scale
• Task adaptation
Instruction Tuning Training
Fine-tuning language models on diverse tasks described via natural language instructions.
Benefits:
• Better instruction following
• Improved zero-shot
• More helpful responses
Interpretability Evaluation
The degree to which humans can understand and explain how an AI model makes decisions or predictions.
Approaches:
• Attention visualization
• Saliency maps
• Layer analysis
• Probing studies
Intelligence Explosion General
Hypothetical scenario where AI systems rapidly self-improve, leading to superintelligence far beyond human capability.
Concept:
• Recursive self-improvement
• Exponential growth
• Theoretical scenario
• Safety concerns

L 20 terms

Large Language Model (LLM) Architecture
Neural networks with billions of parameters trained on vast text data to understand and generate human-like text.
Scale Categories:
• Small: <1B params
• Medium: 1B-10B
• Large: 10B-100B
• Massive: 100B+
Learning Rate Training
Hyperparameter controlling how much to change model parameters during training in response to estimated error.
Effects:
• Too high: Instability
• Too low: Slow convergence
• Range: 1e-5 to 1e-2
LoRA Techniques
Low-Rank Adaptation - parameter-efficient fine-tuning method that reduces trainable parameters.
Advantages:
• Faster training
• Lower memory
• Preserves base model
• Easy adapter swapping
Loss Function Training
Mathematical function that measures difference between predicted and actual outputs, guiding model learning.
Common Types:
• Cross-entropy loss
• Mean squared error
• Binary cross-entropy
• Huber loss

M 18 terms

Multimodal AI Architecture
AI systems that can process and understand multiple types of data (text, images, audio, video) simultaneously.
Examples:
• GPT-4 Vision
• DALL-E 3
• Flamingo
• CLIP
Multi-Head Attention Architecture
Mechanism in Transformers running multiple attention mechanisms in parallel, each focusing on different aspects.
Components:
• Query, Key, Value matrices
• Multiple heads
• Linear projection
• Concatenation
Model Compression Techniques
Techniques to reduce model size and computational requirements while maintaining performance.
Methods:
• Pruning
• Quantization
• Knowledge distillation
• Low-rank factorization
Machine Learning General
Branch of AI that enables computers to learn and improve from experience without being explicitly programmed.
Types:
• Supervised learning
• Unsupervised learning
• Reinforcement learning
• Semi-supervised

N 14 terms

Neural Network Architecture
Computational model inspired by biological neural networks, consisting of interconnected nodes that process information.
Types:
• Feedforward
• Recurrent (RNN)
• Convolutional (CNN)
• Transformer
Next Token Prediction Training
Training objective where model learns to predict the next token in a sequence given previous tokens.
Process:
• Input: "The cat sat on"
• Target: "the"
• Model learns patterns
• Enables generation
NLP (Natural Language Processing) General
Branch of AI focused on interaction between computers and human language, enabling machines to understand, interpret, and generate text.
Applications:
• Language translation
• Sentiment analysis
• Text summarization
• Question answering
Normalization Training
Techniques to standardize inputs to neural network layers, improving training stability and speed.
Types:
• Batch normalization
• Layer normalization
• Group normalization
• Instance normalization

O 8 terms

Optimization Training
Process of adjusting model parameters to minimize loss function and improve performance.
Algorithms:
• Adam
• AdamW
• SGD
• RMSprop
Overfitting Evaluation
When a model learns training data too well, performing poorly on new, unseen data due to memorization rather than generalization.
Prevention:
• Regularization
• Dropout
• Early stopping
• Cross-validation
Open Source General
AI models and code that are publicly available, allowing anyone to use, modify, and distribute them freely.
Examples:
• LLaMA
• Mistral
• Falcon
• Code Llama
OpenAI General
AI research company that developed GPT series, ChatGPT, DALL-E, and other influential AI systems.
Products:
• GPT-4
• ChatGPT
• DALL-E 3
• Codex

P 24 terms

Parameters General
Learnable weights and biases in neural networks that are adjusted during training to minimize loss function.
Scale Examples:
• GPT-3: 175B params
• GPT-4: ~1.7T params
• PaLM: 540B params
Prompt Engineering Techniques
Art and science of crafting effective prompts to get desired outputs from language models.
Techniques:
• Few-shot prompting
• Chain-of-thought
• Role-based prompting
• Template design
Pre-training Training
Initial training phase where models learn general language understanding from large, diverse datasets.
Characteristics:
• Massive datasets
• Unsupervised learning
• General patterns
• Foundation for specialization
Pruning Techniques
Technique to reduce model size by removing weights or neurons that contribute least to performance.
Methods:
• Magnitude-based
• Structured pruning
• Unstructured pruning
• Gradual pruning

T 30 terms

Token General
The basic unit of text that language models process. Can be words, subwords, characters, or other segments depending on the tokenization method.
Examples:
"Hello world!" → ["Hello", " world", "!"]
"ChatGPT" → ["Chat", "G", "PT"] or ["Chat", "GPT"]
Transformer Architecture
Revolutionary neural network architecture that uses self-attention mechanisms, introduced in "Attention Is All You Need" (2017).
Key Components:
• Multi-head attention
• Position encoding
• Feed-forward networks
• Residual connections
• Layer normalization
Temperature Training
A sampling parameter that controls the randomness of model outputs. Higher values increase creativity but may reduce coherence.
Settings Guide:
• 0.0: Deterministic (greedy)
• 0.3: Conservative, focused
• 0.7: Balanced creativity
• 1.0: High creativity
• 2.0: Very random
Top-k Sampling Training
A sampling strategy that considers only the k most likely tokens at each generation step, improving quality by filtering out low-probability options.
Common Values:
• k=1: Greedy decoding
• k=10-50: Focused sampling
• k=100: Broader choices
• Combined with temperature

J 1 term

JSONL Data
JSON Lines format commonly used for storing training data with one JSON object per line.
Usage:
• Dataset storage
• Streaming data
• Model training input

K 1 term

Knowledge Distillation Techniques
Training technique where a smaller "student" model learns from a larger "teacher" model to achieve similar performance with fewer parameters.
Benefits:
• Model compression
• Faster inference
• Lower memory usage

Q 6 terms

Quantization Techniques
Compression technique reducing precision of model weights from 32-bit to lower bit representations (8-bit, 4-bit).
Benefits:
• Reduced memory usage
• Faster inference
• Lower storage requirements
Query Architecture
In attention mechanism, the query vector represents what information is being sought from the sequence.
Attention Components:
• Query (Q)
• Key (K)
• Value (V)
QLoRA Techniques
Quantized Low-Rank Adaptation - efficient fine-tuning method combining quantization with LoRA.
Innovation:
• 4-bit quantization
• LoRA adapters
• Minimal memory usage

R 4 terms

RLHF Techniques
Reinforcement Learning from Human Feedback - training method using human preferences to align model outputs with desired behavior.
Process:
• Collect human preferences
• Train reward model
• Policy optimization
RAG Techniques
Retrieval-Augmented Generation - combining language models with external knowledge retrieval for more accurate responses.
Components:
• Retrieval system
• Knowledge base
• Generator model
RNN Architecture
Recurrent Neural Network - earlier architecture for sequence modeling, largely replaced by Transformers.
Variants:
• LSTM
• GRU
• Bidirectional RNN
Regularization Techniques
Techniques to prevent overfitting by adding constraints or noise during training.
Methods:
• Dropout
• Weight decay
• Data augmentation

S 4 terms

Self-Attention Architecture
Attention mechanism where sequence elements attend to other elements in same sequence to capture dependencies.
Core Innovation:
Enables parallel processing and long-range dependencies
Softmax Techniques
Activation function converting raw scores into probability distribution over vocabulary for token prediction.
Formula:
softmax(x_i) = exp(x_i) / Σ exp(x_j)
Supervised Learning Training
Machine learning approach using labeled data to train models for prediction or classification tasks.
Requirements:
• Input-output pairs
• Ground truth labels
• Loss function
Scaling Laws Training
Empirical relationships showing how model performance improves predictably with scale (parameters, data, compute).
Key Insight:
Performance scales as power law with resources

U 2 terms

Unsupervised Learning Training
Learning patterns from data without explicit labels, foundation of LLM pre-training.
LLM Application:
Next-token prediction from raw text
Upsampling Techniques
Technique to increase resolution or size of data, opposite of downsampling/pooling operations.
Methods:
• Bilinear interpolation
• Transposed convolution
• Nearest neighbor

V 2 terms

Vector Database Techniques
Specialized database optimized for storing and querying high-dimensional vector embeddings.
Examples:
• Pinecone
• Weaviate
• ChromaDB
• Milvus
Validation Set Evaluation
Data subset used during training to tune hyperparameters and monitor performance without overfitting.
Split Ratio:
Typically 10-20% of training data

W 2 terms

Weight Decay Techniques
Regularization technique that penalizes large weights by adding L2 penalty to loss function.
Effect:
Prevents overfitting, improves generalization
Warmup Techniques
Training strategy gradually increasing learning rate from zero to target value during initial steps.
Purpose:
Stable training start, prevents early divergence

X 1 term

Xavier Initialization Techniques
Weight initialization method maintaining variance across layers to prevent vanishing/exploding gradients.
Formula:
Uniform[-√(6/(fan_in+fan_out)), √(6/(fan_in+fan_out))]

Y 1 term

YAML Data
Human-readable data serialization format commonly used for configuration files in ML projects.
Usage:
• Model configs
• Training parameters
• Pipeline definitions

Z 2 terms

Zero-shot Learning Techniques
Model's ability to perform tasks without task-specific training examples, using only instructions.
Example:
"Translate this to French:" without translation training
Z-score Evaluation
Statistical measure indicating how many standard deviations a data point is from the mean.
Formula:
z = (x - μ) / σ
